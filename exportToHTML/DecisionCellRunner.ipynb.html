2    # DecisionCell   
3    A modular multi-agent framework for transparent, tool-augmented decision making The system combines: 
4     
5    - **ReAct-style reasoning** – agents think, take structured actions, call tools, and react to observations   
6    - **Transparent reasoning traces** – each agent exposes its step-by-step thought process   
7    - **Retrieval-Augmented Generation (RAG)** – agents can query their own knowledge bases to ground reasoning in data   
8     
9    This notebook walks you through the full setup required to run a DecisionCell experiment.

19   ## 1. Set up the Staff (agents) 
20    
21   DecisionCell works with a small set of collaborating agents.   
22   Each agent is defined by: 
23    
24   - A **YAML config** (name, role, provider, model, system prompt) 
25   - Optional **tools** attached at runtime (e.g. search, ask_human) 
26   - A shared **logbook** used to record thoughts, actions, tool calls, and observations 
27    
28   In the next cell we create the global logbook,  load the configs, initialize the tools, and create the agents.  
29   #%% 
30   # Create the staff logbook
31   logbook = Logbook()
32   
33   # Create legal agent
34   legal_config = yaml.safe_load(Path("agents/config/LegalAdvisor.yaml").read_text(encoding="utf-8"))
35   
36   search_legal = kb_search("agents/kb/LegalAdvisor", k=3)
37   
38   legal = Agent(
39       config = legal_config,
40       logbook = logbook, 
41       tools =  [ask_human, search_legal],
42   )
43   
44   # Create cyber operations agent
45   cyber_operations_config = yaml.safe_load(Path("agents/config/CyberOperationsExpert.yaml").read_text(encoding="utf-8"))
46   
47   search_cyber_operations = kb_search("agents/kb/CyberOperationsExpert", k=3)
48   
49   cyber_operations = Agent(
50       config = cyber_operations_config,
51       logbook = logbook,
52       tools =  [ask_human, search_cyber_operations],
53   )
54   
55   # Create military operations agent
56   military_operations_config = yaml.safe_load(Path("agents/config/MilitaryOperationsExpert.yaml").read_text(encoding="utf-8"))
57   
58   search_military_operations = kb_search("agents/kb/MilitaryOperationsExpert", k=3)
59   
60   military_operations = Agent(
61       config = military_operations_config,
62       logbook = logbook,
63       tools =  [ask_human, search_military_operations],
64   )
65   
66   # Create commander agent
67   commander_config = yaml.safe_load(Path("agents/config/Commander.yaml").read_text(encoding="utf-8"))
68   
69   ask_legal = agent_as_tool(legal, "ask_legal")
70   ask_cyber_operations = agent_as_tool(cyber_operations, "ask_cyber_operations")
71   ask_military_operations = agent_as_tool(military_operations, "ask_military_operations")
72   
73   commander = Agent(
74       config = commander_config,
75       logbook = logbook,
76       tools =  [ask_human, ask_legal, ask_cyber_operations, ask_military_operations],
77   )

82   ## 2. Define the mission objective 
83    
84   The mission objective describes the operational problem to be analyzed. The objective is expressed as a multi-line natural-language description that 
85   outlines the scenario and available Courses of Action (CoAs).  This text becomes  the first input to the Commander when the decision process begins. 
86   #%% 
87   objective = """  
88   Select the single best Course of Action (CoA) to prevent an imminent attack by the Terrmisous group on a commercial cargo ship carrying chemical agents near the civilian port AricikPortus. 
89    
90   Choose ONE option only (CoA 1, CoA 2, or CoA 3). Do not combine options. 
91    
92   CoA 1: Disable the VicikPortus power grid using previously implanted BlackEnergy3-based malware to prevent loading. 
93   CoA 2: Temporarily neutralize the VicikPortus civilian pump station using a DDoS attack exploiting an unpatched vulnerability, preventing fueling. 
94   CoA 3: Conduct GNSS spoofing to alter the cargo ship’s position, velocity, and heading after departure so it cannot reach its intended target. 
95    
96   """
